Extensions to the OWL-Time Ontology - non-Gregorian calendars

OWL-Time [[owl-time]] is an ontology for temporal entities and relations between them. OWL-Time defines simple temporal entities (intervals and instants). This note adds one new class time:Era, one property time:era and an instance time:CommonEra to allow for the description of eras; as well as two datatypes time:generalDayL and time:generalMonthL to allow for the description of days and months in calendar systems with leap and omitted ones.

The namespace for OWL-Time terms is http://www.w3.org/2006/time#

The suggested prefix for the OWL-Time namespace is time

The OWL-Time ontology is available here.

The OWL-Time ontology with these extensions is available here.

New classes and properties are introduced in this extension to OWL-Time. The new elements concern temporal entities which cannot be expressed in OWL-Time. The ontology document imports OWL-Time, and adds the new elements and axioms. IRIs denoting the new elements are placed in the OWL-Time namespace since it is expected that they could be added to the normative ontology in a future revision of OWL-Time.

Motivation and background

OWL-Time is an ontology for describing temporal information. As part of its mission it implements way to encode dates in the Gregorian calendar.

This extension answers a need by Digital Humanists to encode dates in other types of calendar, having features such as eras, leap days and leap month. Such examples include most luni-solar calendars found in Asia.

Vocabulary specification

Classes

Era

Class: time:Era
IRI: http://www.w3.org/2006/time#Era
Definition: A linear (non-cyclic) year numbering system. No assumption is made with regards to the details of the numbering such as the presence of a year 0, or the possibility of years numbered negatively. Any year numbering system can be an instance of this class even if the year definition doesn't exactly match the year definition of the Gregorian calendar. For instance Roman Emperor regnal years or Chinese eras can be encoded as instances of :Era.
Instance of: owl:Class

Datatypes

generalDayL

Datatype: time:generalDayL
IRI: http://www.w3.org/2006/time#generalDayL
Definition: Day of month in a calendar system where they can be leap or omitted - formulated as a text string with a pattern constraint to reproduce a lexical form similar to time:generalDay. The difference with the latter is an optional suffix at the end, in the form of one letter, which can be:
  • f to represent the first day of a leap day
  • s to represent the second day of a leap day
  • o to represent an omitted day (and thus an invalid date)
The value-space is not defined, so a generic OWL2 processor cannot compute ordering relationships of values of this type. The suffix is not mandatory even in the case of a leap day; this allows the representation of dates found in historical documents where ambiguity is present.
Instance of: rdfs:Datatype
Subclass of: owl:onDatatype xsd:string ; owl:withRestrictions ( [ xsd:pattern "---(0[1-9]|[1-9][0-9])(Z|(\\+|-)((0[0-9]|1[0-3]):[0-5][0-9]|14:00))?[fso]?"^^xsd:string ; ] )

generalMonthL

Datatype: time:generalMonthL
IRI: http://www.w3.org/2006/time#generalMonthL
Definition: Month of year in a calendar system where they can be leap or omitted - formulated as a text string with a pattern constraint to reproduce a lexical form similar to time:generalMonth. The difference with the latter is an optional suffix at the end, in the form of one letter, which can be:
  • f to represent the first month of a leap month
  • s to represent the second month of a leap month
  • o to represent an omitted month (and thus an invalid date)
The value-space is not defined, so a generic OWL2 processor cannot compute ordering relationships of values of this type. The suffix is not mandatory even in the case of a leap month; this allows the representation of dates found in historical documents where ambiguity is present.
Instance of: rdfs:Datatype
Subclass of: owl:onDatatype xsd:string ; owl:withRestrictions ( [ xsd:pattern "---(0[1-9]|[1-9][0-9])(Z|(\\+|-)((0[0-9]|1[0-3]):[0-5][0-9]|14:00))?[fso]?"^^xsd:string ; ] )

Examples

Dates with leap days and months in a Tibetan calendar

September 25th 1994 is found in the Phugpa Tibetan calendar as being the second day numbered 20th is the first of two months numbered 8. This would be noted: 

ex:Sept25thPhugpa
  a time:GeneralDateTimeDescription ;
  rdfs:label "September 25th in the Phugpa Tibetan calendar" ;
  time:month "--8f"^^time:generalMonthL ;
  time:day "---20s"^^time:generalDayL ;
  ...
.
